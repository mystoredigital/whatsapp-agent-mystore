    Webhooks salientes

    El server hace POST al URL registrado cuando ocurren eventos. Cada delivery lleva header X-Webhook-Signature: sha256=<hex> donde hex = HMAC-SHA256(secret, body). El receptor recalcula y compara para validar autenticidad. 2 reintentos con backoff (2s, 8s).
